# 引用意图分析证据协议

在写出 `analysis.md` 和最终确认 `citation_graph.json` 之前，先按本协议整理关键引用的证据链。目标不是只给引用贴标签，而是说明“这条引用如何帮助理解目标论文的核心问题、方法、实验或局限”。

## 证据来源

只能使用用户提供的 PDF 文本、已抽取全文、引用上下文、参考文献列表或用户明确给出的笔记。不得用领域常识、记忆或看似合理的参考文献信息补全缺口。

每条关键 citation 尽量记录以下证据锚点：

| 证据锚点 | 含义 |
| --- | --- |
| `citation_context` / 引用上下文 | 引用出现处的句子或短段落，保持简短。 |
| `section` / 章节位置 | 引用附近可见的章节名、小节名或表格位置。 |
| `target_claim` / 目标论文论点 | 该引用支撑的目标论文问题、方法选择、数据选择或结果解释。 |
| `cited_work_role` / 被引工作角色 | 被引工作的作用，例如问题来源、方法组件、数据集来源、基线、工具、理论、结果证据或局限。 |
| `intent_rationale` / 意图判据 | 为什么该引用应归入当前 intent，而不是相邻标签。 |
| `confidence_reason` / 置信度理由 | 为什么置信度是高、中或低。 |

## 引用上下文规则

- 优先使用引用句及其相邻上下文，不要只根据摘要或论文常识判断。
- 如果引用只出现在表格、图注或 PDF 抽取噪声中，应在 `notes` 中说明噪声来源，并视情况降低置信度。
- 如果无法可靠匹配参考文献条目，应设置 `unmatched_reference: true`，并说明不确定性。
- `analysis.md` 中不要大段复制原文；只保留短证据片段，并用中文解释其作用。

## 意图判据规则

每条关键 citation 都应能回答：

1. 它支撑目标论文的哪个论点或设计选择？
2. 被引工作在该论点中扮演什么角色？
3. 为什么选择当前 `intent`，而不是相近标签？
4. 可见证据对该判断的支撑强度如何？

常见判据：

- `core-method`：被引工作直接支撑目标论文的核心方法组件或关键设计。
- `supporting-method`：被引工作支撑辅助技术、实现选择或背景机制。
- `dataset`：被引工作定义或引入目标论文使用的数据集。
- `baseline`：被引工作作为比较系统或对照方法，而不只是相关工作。
- `result-evidence`：被引工作或外部模型结果用于解释目标论文的实验效果。

## 置信度规则

`confidence` 使用以下标准：

| 等级 | 数值范围 | 使用条件 |
| --- | ---: | --- |
| 高 | `0.80-1.00` | 引用句、参考文献匹配和目标论文论点都清楚。 |
| 中 | `0.55-0.79` | intent 基本可靠，但章节、参考文献匹配或被引角色有一项不完整。 |
| 低 | `0.10-0.54` | 证据来自噪声文本、表格残片、模糊上下文，或与目标论点连接较弱。 |

只有参考文献标题、没有引用上下文时，不得给高置信度。

## 无证据处理

如果某个预期 intent、方法链、数据集链或基线链没有可靠证据，必须写 `未发现可靠证据`。不要为了填满模板表格而编造引用。

生成 `analysis.md` 时，要明确区分：

- 有证据支撑的结论；
- 合理但不确定的解释；
- 缺失、噪声或无法匹配的证据。

生成 `citation_graph.json` 时，应把不确定性写入 `notes`、`intent_rationale` 或 `confidence_reason`，不要隐藏不确定性。
