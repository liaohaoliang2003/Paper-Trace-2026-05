# 中心论文放射图视觉规范

默认可视化是一张中心论文放射图：目标论文作为主节点，不同语义分组围绕它展开，引用、参考文献和实体节点向外分支。只要 SVG 生成可用，应将该图保存为 `citation_map.svg`。如果无法生成 SVG，则保存 `citation_map_spec.md`，记录同等布局决策，并保持 `citation_graph.json` 完整。

## 视觉目标

图谱需要展示目标论文如何借助已有问题、方法、数据集、指标、基线和证据构建自身贡献。它应该像“视觉摘要”，而不是拥挤的参考文献网络。

## 布局

默认画布：

| 属性 | 默认值 |
| --- | --- |
| 宽度 | `1400` |
| 高度 | `1000` |
| 中心节点 | 右侧居中，坐标约为 `(980, 500)` |
| 左侧分组 | 问题、方法、评估、基线/结果 |
| 底部分组 | 局限/未来，可选 |

推荐分组位置：

| 分组 | 位置 | 作用 |
| --- | --- | --- |
| 问题背景 | 左上 | 背景、问题和理论引用 |
| 核心方法 | 左中 | 核心方法和辅助方法引用 |
| 数据与评估 | 中上或中部 | 数据集和评价指标引用 |
| 基线与结果 | 左下 | 基线方法和结果证据 |
| 局限与未来 | 底部或更左侧 | 局限和未来工作 |

如果 citation 节点超过 25 个，SVG 只展示高优先级节点，其余引用用数量徽标汇总。

## 节点类型

| 节点 | 形状 | 样式 |
| --- | --- | --- |
| 目标论文 | 大圆角矩形 | 浅蓝灰背景、深色文字 |
| 分组枢纽 | 圆角矩形 | 白底、彩色描边 |
| 实体 | 小圆角矩形 | 使用同组浅色填充 |
| 参考文献 | 文本标签或胶囊节点 | 灰色文字，可显示作者-年份 |
| 证据注释 | 小号旁注 | 弱化灰色 |

## 边类型

| 关系 | 边样式 |
| --- | --- |
| 直接引用意图 | 从目标论文到分组枢纽的实线曲线 |
| citation 派生实体 | 从分组枢纽到实体的短实线曲线 |
| 间接支持或次要意图 | 虚线曲线 |
| 不确定匹配 | 低透明度点线 |

箭头应谨慎使用。主要阅读方向是从目标论文向外理解其引用支撑。

## 意图颜色

| 意图组 | 颜色 |
| --- | --- |
| 问题/背景 | `#cf6f6f` |
| 核心/辅助方法 | `#ef6c2f` |
| 数据集/指标 | `#8a5cf6` |
| 基线/结果 | `#d18a19` |
| 局限/未来 | `#4f9c56` |
| 未匹配/不确定 | `#9aa3ad` |

## SVG 结构

推荐 SVG 层级：

```xml
<svg width="1400" height="1000" viewBox="0 0 1400 1000" xmlns="http://www.w3.org/2000/svg">
  <defs>
    <!-- gradients, shadows, arrowheads -->
  </defs>
  <rect width="1400" height="1000" fill="#fbfbfa"/>
  <g id="edges">
    <!-- 先绘制曲线边，保证节点覆盖在边上方 -->
  </g>
  <g id="groups">
    <!-- 分组枢纽和组内节点 -->
  </g>
  <g id="target-paper">
    <!-- 中心目标论文节点 -->
  </g>
  <g id="legend">
    <!-- 意图颜色图例 -->
  </g>
</svg>
```

最小 SVG 内容：

- 一个可见的目标论文节点，标签使用论文标题。
- 至少一个来自 `visual_groups` 的分组 hub。
- 对每个有数据的高优先级分组，至少展示一个 citation、entity 或 reference 节点。
- 从目标论文到分组 hub、从分组 hub 到展示节点的曲线边。
- 显示颜色与 intent 分组对应关系的图例。
- 使用来自 `citation_graph.json` 的短证据标签或节点副标题。

## 标注规则

- 目标论文标签：使用标题；超过 60 个字符时缩短。
- 分组枢纽标签：使用中文语义标签，例如 `核心方法`。
- 参考文献标签：优先使用 `Short Title (Author, Year)`；未匹配时使用 `unknown reference [marker]`。
- 边标签：使用简短中文意图，例如 `核心方法`、`数据集`、`评价指标`。
- 证据旁注应使用短语，不放完整长句。

## 渲染策略

首版 CLI 渲染器应采用确定性 SVG 生成：

1. 按默认顺序排序分组。
2. 组内节点按优先级排序：`core-method`、`dataset`、`metric`、`baseline`，再按置信度降序。
3. 将分组枢纽放在固定锚点坐标。
4. 组内节点围绕枢纽垂直排列。
5. 使用三次贝塞尔曲线连接目标论文、分组枢纽和节点。

V1 不使用随机力导向布局，因为随机布局不利于复现和测试。

## 图谱简化

当图谱过密时：

- `citation_graph.json` 保留全部记录。
- 每个可视化分组只渲染前 3 到 5 个节点。
- 添加类似 `+12 background citations` 的分组数量徽标。
- 多条 citation 支撑同一概念时，优先展示概念/实体节点，而不是所有参考文献节点。

## 可访问性

- 保持文字与背景有足够对比。
- 同时使用颜色和文字标签，不只依赖颜色。
- 包含图例，说明颜色和意图分组的对应关系。
- 主标签字号至少 `16px`，次级标签至少 `13px`。
